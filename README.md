# سينما بلس · CinemaPlus

## نظرة عامة على المشروع
- **الاسم**: سينما بلس (CinemaPlus)
- **الهدف**: منصة عربية لمشاهدة الأفلام والمسلسلات والأنمي مترجم اون لاين بجودة عالية HD/4K، بتصميم سينمائي حديث ونظام ثيمات متعدد.
- **الدومين**: `https://cinemaplus.site`
- **المكتبة**: 5814 عملاً (2915 فيلم · 737 مسلسل · 2162 أنمي).

## أهم المزايا المنفّذة
- ✅ **إعادة علامة تجارية كاملة** إلى سينما بلس / CinemaPlus عبر كل النصوص والبيانات والـ JSON-LD.
- ✅ **روابط مشفّرة بالكامل (URL Obfuscation)** — لا تحتوي المسارات على كلمات `movie/series/anime/watch` صريحة:
  - أفلام → `/f/{slug}` · مسلسلات → `/d/{slug}` · أنمي → `/n/{slug}`
  - القوائم → `/x/f` · `/x/d` · `/x/n` (مع ترقيم `/p/{رقم}`)
  - صفحة المشاهدة (البوابة) → `/g/{token}` حيث الـ token ترميز XOR + base64url عكسي للـ slug.
- ✅ **الحفاظ التام على مسارات الصور/البوسترات** (`/images/{category}/...`) — لم يُمسّ حقل `poster` إطلاقاً.
- ✅ **نظام ثيمات متعدد** عبر CSS Custom Properties وخاصية `data-section`:
  - الأنمي: سماوي/بنفسجي نيون · الأفلام: ذهبي/قرمزي سينمائي · المسلسلات: زمردي/برتقالي درامي · الرئيسية: مزيج يتبدّل مع شرائح الـ hero.
- ✅ **تصميم UI/UX جديد**: hero سلايدر يبدّل الثيم لكل شريحة، صف Trending TOP 10، أقسام مصنّفة، glassmorphism، توهّج نيون، scroll rails.
- ✅ **سيو عربي قوي**: Meta + OpenGraph (ar_AR)، بيانات منظّمة JSON-LD (WebSite / Movie / TVSeries / TVSeason / TVEpisode)، canonical لكل الصفحات، وrobots/sitemap مضبوط للفهرسة.
- ✅ **robots.txt و sitemap** محدّثان إلى الدومين الجديد، مع استبعاد بوابة المشاهدة `/g/` من الفهرسة.

### تحديثات فحص SEO والفهرسة (آخر جلسة)
- ✅ **فحص شامل للنصوص المرئية** في ملفات الأعمال والقوائم والصفحات المبنية: لا توجد عبارات علامة قديمة أو عبارات مصدر بيانات ظاهرة للمستخدم.
- ✅ **4739 من 5814 عملاً (≈81.5%)** لديها ملخصات قصة فعلية وفريدة؛ وبقية الأعمال تحصل على وصف عربي احتياطي غير فارغ لتجنب الصفحات الرقيقة.
- ✅ **إزالة كل العبارات المخالفة من الصفحات العامة**: لا تظهر عبارات مصدر البيانات أو الترويج القالبي في HTML الناتج.
- ✅ **ترتيب موثوق آمن للسيو** في قوائم الأفلام والمسلسلات والأنمي:
  - «مختارة» (افتراضي، يدفع الأعمال ضعيفة المصداقية والحلقات الخاصة للأسفل) · «الأحدث» · «الأعلى تقييماً» · «أبجدي».
  - التقييم يعتمد صيغة **Bayesian (IMDb-style)**: `(v/(v+m))·R + (m/(v+m))·C` بـ m=50, C=6.5 — لا يسمح لتقييم 9.5 بثلاثة أصوات أن يتفوّق على 8.4 بآلاف الأصوات.
- ✅ **معالجة الحلقات الخاصة (الأنمي)**: تمييز `is_special` للحلقات الخاصة/OVA/Recap ودفعها لآخر القوائم بدل تصدّرها أبجدياً.
- ✅ **إصلاح روابط مشاهدة الحلقة الواحدة/OVA/Special**: أزرار المشاهدة وتبويبات الحلقات للأعمال التي لا تُبنى لها صفحة حلقة ثابتة تنتقل الآن مباشرة إلى بوابة المشاهدة `/g/{token}?s={season}&e={episode}` بدل رابط حلقة غير موجود مثل `/n/{slug}/c/1/e/1`.
- ✅ **تصنيفات أنمي ثقافية**: تفكيك الرموز المركّبة مثل «حركة ومغامرة» وتطبيع وترتيب ثقافي (أنمي يتصدّر، ثم شونين/سينين/إيسيكاي…) عبر `GENRE_SPLIT` + `orderAnimeGenres`.

## معمارية البيانات
- **مصدر البيانات**: ملفات JSON لكل عمل في `src/data/titles/*.json` → يجمعها `scripts/build-data.mjs` إلى `src/data/generated/all.json`.
- **نموذج البيانات**: `Title` (slug, clean_title, category, poster, seasons[], episodes[], servers[], genre, year, quality, language ...).
- **ترميز الروابط**: مركزي في `src/lib/routes.ts` (detail/listing/season/episode + encode/decodeGateToken).
- **تحميل العناوين**: `src/lib/titles.ts` يعيد كتابة حقل `url` فقط إلى الشكل المشفّر (دون لمس الصور).
- **محتوى السيو/الوصف**: القصة الفعلية أولاً، ثم مولّدات نصّية حتمية عربية في `src/lib/detailContent.ts` للفقرات السياقية عند نقص البيانات، مع وصف احتياطي في `scripts/build-data.mjs` لتجنب المحتوى الرقيق.
- **الإثراء والترتيب**: كاش داخلي في `scripts/.cache/` يدمجه `build-data.mjs` ويولّد `sort_rating` (Bayesian) و`sort_recent` و`is_special` و`real_plot` دون إظهار أي نسب مصدر داخل الصفحات العامة.
  - سكربتات الإثراء تحتاج مفاتيح API في `.dev.vars` عند استخدامها محلياً.
- **حقول الترتيب**: `Title.{tmdb_vote,tmdb_votes,release_date,sort_rating,sort_recent,real_plot,is_special}` + نفسها في `TitleIndexEntry` لاستخدامها في الفرز من جهة العميل.

## دليل الاستخدام السريع
1. تصفّح الأقسام من القائمة العلوية (أفلام/مسلسلات/أنمي) — يتغيّر الثيم تلقائياً حسب القسم.
2. افتح صفحة عمل لرؤية القصة والمعلومات والأعمال المشابهة، ثم اضغط «شاهد الآن».
3. في بوابة المشاهدة بدّل بين السيرفرات والحلقات، أو استخدم «السابق/التالي».
4. استخدم `/search` للبحث الفوري الضبابي (fuse.js).

## التطوير والبناء
```bash
npm run data          # توليد البيانات (يعالج ترميز أسماء الملفات العربية + تنظيف العلامة)
npm run build         # بناء Astro الثابت إلى dist/
pm2 start ecosystem.config.cjs   # تشغيل خادم المعاينة الثابت على المنفذ 3000
```

### ملاحظة المعاينة (هام)
بيئة المعاينة محدودة الذاكرة (~1GB)، وبناء 5800 عمل يولّد عشرات آلاف الصفحات ويتجاوز الذاكرة. لذلك:
- `src/data/generated/all.json` محفوظ حالياً بالكتالوج الكامل بعد تشغيل `npm run data` فقط، دون بناء كل الصفحات.
- يمكن اختبار البناء محلياً بمجموعة صغيرة عبر `PREVIEW_LIMIT=60 npm run build` أو قيمة قريبة، ثم إعادة `npm run data` لاسترجاع `all.json` الكامل.
- **النسخة الكاملة الاحتياطية (5814 عملاً، 93MB تقريباً) محفوظة في** `src/data/generated/all.full.json`.
- **للنشر الإنتاجي الكامل**: ابنِ على بيئة CI ذات ذاكرة أكبر بدلاً من sandbox محدود الذاكرة.
